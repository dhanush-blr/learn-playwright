# Source Code vs Byte Code vs Binary Code

## Example: `01_HelloWorld.js`

```javascript
console.log("Hello, Dhanush!");
```

## Comparison Table

| Aspect | Source Code | Byte Code | Binary Code (Machine Code) |
|--------|-------------|-----------|----------------------------|
| **What it is** | Human-readable instructions written in a programming language | Intermediate representation between source and machine code — platform-independent | CPU-native instructions — 1s and 0s executed directly by the processor |
| **Example** | `console.log("Hello, Dhanush!");` | `get_field r1, "console"`<br>`get_property r2, r1, "log"`<br>`call r2, ["Hello, Dhanush!"]` | `48 8b 05 f8 3b 00 00`<br>`48 89 c7 48 8d 35 1c 00 00 00`<br>`31 c0 e8 e9 fc ff ff` |
| **Readable?** | Yes — written by and for humans | Partially — mnemonics but not meant for human consumption | No — raw hex/binary, only readable by the CPU |
| **Generated by** | Developer writes it in an editor | Compiler/Interpreter (V8's Ignition for JS) | Assembler or JIT compiler (V8's TurboFan for hot JS paths) |
| **Stored in** | `.js` file on disk | In memory at runtime (or in `.class`/`.pyc` for compiled langs) | On disk as an executable binary (`.exe`, `.out`, ELF) |
| **Execution** | Fed to a runtime (Node.js) that parses it | Interpreted or JIT-compiled to native code | Runs directly on the CPU with zero translation |
| **JS-specific** | What you write in your editor | V8 compiles this internally — never saved to a file | V8's TurboFan emits x86/ARM machine code for hot paths at runtime |

## Real-world flow for this example

When you run `node 01_HelloWorld.js`:

1. **Source code** → Node.js reads the `.js` file
2. **Parsing** → V8 parses source into an AST (Abstract Syntax Tree)
3. **Byte code** → V8's Ignition interpreter generates byte code from the AST
4. **Binary code** → If `console.log` is called frequently, V8's TurboFan JIT compiles it to native machine code for faster execution

> **Note**: For JavaScript, byte code and binary code exist only **in memory at runtime** — they are never written to disk as files.
